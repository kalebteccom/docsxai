<div class="docsx-section not-content">
  <h2 class="docsx-section-title">Why it exists</h2>
  <div class="docsx-section-sub">
    Screenshot docs rot the moment the UI moves. docsxai treats them as a
    build artifact: a written flow is the source, the rendered doc pack is
    the output, and a re-run is the refresh.
  </div>

  <div class="docsx-grid">
    <div class="docsx-card">
      <span class="docsx-card-mark">// execution</span>
      <h3>Deterministic CI replay</h3>
      <div class="docsx-card-text">
        A frozen clock, pinned locale, timezone, and viewport: the same flow
        against the same target state produces byte-identical screenshots.
        Baseline the pack, gate CI on pixel-level drift with
        <code>diff --fail-on warn</code>.
      </div>
    </div>
    <div class="docsx-card">
      <span class="docsx-card-mark">// calibration</span>
      <h3>Agent calibration, any MCP host</h3>
      <div class="docsx-card-text">
        An agent authors the flow-file once, through the Claude Code plugin
        or the standalone MCP server, with lint, diagnose, and actionability
        probes at write-time. After that, every run is agent-free: no model
        in the loop, no token bill per refresh.
      </div>
    </div>
    <div class="docsx-card">
      <span class="docsx-card-mark">// plugins</span>
      <h3>A plugin runtime with teeth</h3>
      <div class="docsx-card-text">
        Publishers, renderers, lint rules, and auth strategies extend the
        engine through a namespaced, capability-declared, sha256-locked
        runtime. Confluence and Starlight ship as first-party plugins.
      </div>
    </div>
    <div class="docsx-card">
      <span class="docsx-card-mark">// output</span>
      <h3>Docs as code, rendered three ways</h3>
      <div class="docsx-card-text">
        One doc pack, three deliveries: an interactive viewer with halos and
        callouts, a production Astro Starlight site, or burned-annotation
        PNGs pushed to Confluence as ADF. Redactions mask sensitive UI
        before any pixel hits disk.
      </div>
    </div>
  </div>
</div>
